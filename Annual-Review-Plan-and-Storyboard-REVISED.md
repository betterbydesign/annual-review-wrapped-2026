# Scott Foster — Annual Review “Wrapped”
## Revised Browser-Based Plan, Metric System & Copy Direction

*Prepared for the July 2025 → June 2026 review window.*  
*Revision goal: make the experience feel like a browser-native Wrapped story, not a slide deck. One idea per screen. Percent-first metrics. Short copy. Motion does the explaining.*

---

## 1. What changed from the prior draft

The previous files had strong substance, but too much of the copy read like a leadership memo placed onto slides. This revision tightens the story for a shareable, browser-based experience.

**Rewrite rules used throughout:**

- Lead with the **outcome**, not the backstory.
- Convert before/after time claims into **percent improvements**.
- Keep each story screen to **one main number + one plain-English caption**.
- Move details, caveats, and proof into **notes**, not on-screen text.
- Use visual categories, motion, and iconography instead of section-heading slides.
- Treat revenue/margin details as **leadership-only** unless the audience is limited.

---

## 2. Design inspiration to borrow from Spotify Wrapped

This should be **Wrapped-inspired**, not a Spotify clone. Use Altitude colors, Agent Hub UI motifs, and your own metrics, but borrow the storytelling mechanics.

**Borrow these patterns:**

1. **Big stat first.** Wrapped turns data into identity: oversized numbers, short labels, and emotional framing.
2. **One card = one thought.** Avoid multi-bullet slides. Each scene should be scannable in 3–5 seconds.
3. **Bold contrast.** Use intense, unexpected color pairings from the Altitude palette: Navy + Summit, Red Rocks + Basecamp, Alpine + Navy, Blue + Basecamp.
4. **Layered “share cards.”** Use stacked rectangles, rounded panels, stickers, chips, texture, and geometric bursts.
5. **Micro-story progression.** The viewer should feel a journey: speed → systems → protection → recognition → what’s next.
6. **Social proof as a moment.** Quote cards should feel like collectible Wrapped cards, not a testimonials slide.
7. **Motion over explanation.** Countdowns, progress fills, tile flips, orbiting chips, and stacked-card reveals should replace long setup copy.

**Avoid these patterns:**

- Agenda slides with six equal bullets.
- Paragraphs inside bordered boxes.
- Long quote blocks.
- Multiple metrics competing on one screen.
- Formal PowerPoint section labels like “Theme 3a.”
- Sensitive pricing or margin data in a broad coworker-share version.

## 4. Metric translation system

Use this formula for any time-saved stat:

> **Efficiency gain / time reduction = (old time − new time) ÷ old time**

### Rewritten hero metrics

| Original metric | Percent-first version | Use on screen? | Notes |
|---|---|---:|---|
| 5–10 hrs → ~30 min prototype build | **90–95% faster prototypes** | Yes | Caption can include “5-page prototype: 5–10 hrs → ~30 min.” |
| 2 weeks → 2 days full build + QA | **~80% faster site builds** | Yes | Uses 10 workdays → 2 workdays. If using calendar days, it is ~86%; use the more conservative 80%. |
| Security packages at 95.2% / 92.9% gross margin | **93–95% gross margin** | Leadership-only | For coworker-safe version, use “Protection program productized across 29 sites.” |
| RCA rate $250/hr → $400/hr | **+60% billable-rate reset** | Leadership-only | Good leadership metric; not ideal for broad coworker sharing. |
| Wordfence upsell +$4,140/yr profit | **+$4.1k profit per upsell** | Leadership-only | Use in notes or leadership version only. |
| BigID $540/yr × 13 enrolled | **$7k new recurring line** | Maybe | Safe if revenue numbers are okay for the audience; otherwise say “new recurring consent-management add-on.” |
| ≥8 malware incidents contained | **≥8 incidents contained** | Yes | Strong, simple, and understandable. |
| 14-stage QA framework | **14-stage QA system** | Yes | Do not overexplain each stage on screen. |
| QA goal to improve by 50% | **50% fewer QA misses target** | Yes, marked target | Do not imply achieved unless measured. |
| 63,000-row importer | **63k-row metadata importer** | Yes/secondary | Great quick proof of technical scale. |
| 90 pages / 62 products migrated | **152 migration objects automated** | Maybe | “90 pages + 62 products” is clearer than 152 unless animated. |
| 182 commits on Apex Studio | **182 commits** | Maybe | Better for dev audience. For broader viewers, “278 tests passing” may read as stronger proof. |
| 9 #props since July | **9 shoutouts since July** | Yes | Use as the social-proof scene. |

### Recommended hero stat set

Use six stat cards max:

1. **90–95% faster prototypes**  
   *5-page prototype: 5–10 hrs → ~30 min.*
2. **~80% faster site builds**  
   *Full build + QA: 2 weeks → 2 days.*
3. **≥8 incidents contained**  
   *Malware response, root cause, client RCAs.*
4. **14-stage QA system**  
   *Quality gates from kickoff to launch.*
5. **29 protected sites**  
   *Managed protection program productized.*
6. **9 org shoutouts**  
   *CEO, SVPs, revenue, and client-facing teams.*

**Leadership-only alternate hero stat:** swap **29 protected sites** for **93–95% gross margin**.

## 5. Copy system

### Headline rules

- 4–8 words preferred.
- Maximum 11 words.
- No “Theme,” “Section,” or “Strategic Vision & Risk” labels on-screen.
- Use active verbs: cut, built, protected, shipped, turned, systemized.

### Caption rules

- One sentence.
- 12–18 words max.
- Put raw before/after data in the caption, not the headline.
- Never put more than three facts on a card.

### Better copy examples

| Long version | Wrapped-style version |
|---|---|
| Figma→WordPress AI pipeline: prototype creation 5–10 hours → ~30 minutes for a 5-page site. | **90–95% faster prototypes.** 5-page prototype: 5–10 hrs → ~30 min. |
| Headless / WordPress-exit strategy — targeting 30% less QA and 80% fewer maintenance/security issues. | **A cleaner web stack is next.** Target: 30% less QA, 80% fewer maintenance/security issues. |
| Led the malware-incident response wave — ≥8 client sites contained, root-caused, and documented. | **≥8 incidents contained.** Sites cleaned, causes documented, clients reassured. |
| Authored 14-stage QA & Handoff stage-gate framework. | **Quality got a system.** 14 stages, each with an exit gate. |
| Recognition came from across the org — and from clients. | **The work got noticed.** 9 shoutouts since July. |

## 6. Content hierarchy

### On-screen content

Use only:

- Hero numbers.
- Short captions.
- Short quote excerpts.
- Names/titles for attribution.
- Tiny proof chips, when needed.

### Notes / hidden metadata

Move these off-screen:

- Package pricing.
- Margin math.
- Full source confidence labels.
- Long project descriptions.
- Org/accountability wording.
- Caveats like “confirm baseline.”
- Detailed tool names unless the audience needs them.

## 7. Audience-safe versioning

Because this may be shared with coworkers, make the default experience **coworker-safe**.

### Coworker-safe

- ~80% faster builds.
- ≥8 incidents contained.
- 14-stage QA system.
- 29 protected sites.

Avoid:

## 8. Visual system

### Format

Build as a browser-native story reel:

- Full viewport scenes.
- Responsive desktop-first with mobile-safe 9:16 share-card crops.
- Auto-advance default: 8–12 seconds per scene.
- Manual controls: arrows, click/tap, progress bar.
- “Pause” control for live walkthroughs.
- Exportable scene PNGs for Slack sharing.

### Layout patterns

Use 5 reusable card types:

1. **Hero stat card** — giant percentage/number, tiny caption.
2. **Before/after timer** — old time collapses into new time.
3. **System map** — chips orbit or connect into one reusable workflow.
4. **Incident grid** — site tiles flip from Red Rocks to Alpine.
5. **Quote card stack** — one short quote at a time, with attribution.

### Motion motifs

- Number flips / odometer counts.
- Progress fills.
- Stacked cards sliding apart.
- Colored chips snapping into a grid.
- Site tiles changing state.
- Quote cards rotating in place.
- Altimeter needle sweep only at open and close.

### Color guidance

Use Altitude tokens, but with Wrapped energy:

| Moment | Suggested palette | Use |
|---|---|---|
| Opening / close | Navy + Summit | Calm, credible, branded. |
| Speed wins | Alpine + Navy + Basecamp | Bright, high-energy. |
| AI/toolkit | Blue + Navy + Summit | Technical but optimistic. |
| Revenue/protection | Basecamp + Navy + Alpine | Business value without looking like finance charts. |
| Security | Red Rocks → Alpine | Threat → contained. |
| Recognition | Navy + Trailhead + Summit | Warm, human, quote-forward. |
